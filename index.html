How it works
Encryption
A random base key is generated client-side. A master key is derived from it using PBKDF2 + SHA-256 (and your optional password). Your note is then encrypted with AES-256-GCM.
Zero knowledge
The base key is only ever in the URL fragment — never sent to a server. Even if someone reads your localStorage, all they see is ciphertext. Without the key, it's meaningless.

Encryption steps
01 — Generate
Random 256-bit base key generated in your browser using Web Crypto.
02 — Derive
Master key derived via PBKDF2 + SHA-256 from base key + optional password.
03 — Encrypt
Note encrypted with AES-256-GCM. IV is random and stored alongside ciphertext.
04 — Share
Base key lives only in the URL fragment — never touches a server or localStorage.
